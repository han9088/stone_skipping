# 📄 PRD：打水漂体感模拟器（Stone Skip Simulator）— Godot 4 版

> 产品需求与 [PRD.md](./PRD.md) 一致，本文档为 **Godot 4** 技术实现版本。  
> 学习路线见 [Godot4-Learning-Roadmap.md](./Godot4-Learning-Roadmap.md)。

---

## 1. 产品概述

### 1.1 产品名称

Stone Skip / 打水漂模拟器（暂定）

### 1.2 产品定位

基于手机体感（加速度 + 陀螺仪）+ 3D 回放的沉浸式投掷模拟应用。

用户通过「模拟打水漂动作」完成一次投掷，系统生成 3D 水面打水漂回放动画。

### 1.3 支持平台

| 平台 | 最低版本 | 说明 |
|------|----------|------|
| **iOS** | iOS 14+ | iPhone（需加速度计 + 陀螺仪） |
| **Android** | Android 8.0 (API 26)+ | 主流机型（需加速度计 + 陀螺仪） |

- V1 同步支持 **Android** 与 **iOS** 双端发布
- 采用 **Godot 4 单项目**，GDScript 为主（可选 C#）
- 平台差异体现在 `Input` 传感器轮询、导出预设、震动权限

---

### 1.4 核心体验

用户无需持续观看屏幕，核心分 **两个阶段**：

**阶段一 · 投掷与体感（主玩法）**

甩动手机 → 识别「出手」→ 实时模拟水漂 → 每个落点震一次 → 水漂结束展示结果

**阶段二 · 3D 回放（可选）**

水漂全部结束后，用户主动进入 3D 回放，重温轨迹与水面特效。

> 震动发生在 **扔出之后的实时模拟阶段**，不在 3D 回放中重复震动。

---

## 2. 核心玩法

### 2.1 用户流程

#### 阶段一：投掷 + 实时反馈（无需看屏幕）

1. 手持手机（模拟石头，**不可真的扔出手机**）
2. 做打水漂甩动动作
3. 系统识别「出手瞬间」，提取投掷参数
4. **实时**运行物理模拟，按落点时间线依次震动
5. 水漂结束（石头下沉），展示结果（次数 / 评分）

#### 阶段二：3D 回放（可选，水漂结束后）

6. 用户点击「查看回放」
7. 播放 3D 水面打水漂动画（镜头 / 水花 / 慢动作）
8. 回放纯视觉，**不再触发震动**

---

### 2.2 体验时序

```
[待机] → [甩动] → [识别出手] → [实时模拟 + 落点震动 × N] → [结果页] → [可选：3D 回放]
         ↑ 用户操作      ↑ 难点              ↑ 主反馈                    ↑ 纯视觉
```

## 3. 核心功能模块

### 3.1 动作采集模块（Motion Capture）

#### 输入设备

- 加速度计（Accelerometer）
- 陀螺仪（Gyroscope）

#### 功能

- 在 `_physics_process` 中高频轮询（目标 60Hz+，尽力接近 100Hz）
- 记录最近 1~2 秒动作数据（环形缓冲区）
- 捕捉运动峰值
- 跨平台统一数据格式

#### Godot 实现

| API | 说明 |
|-----|------|
| `Input.get_accelerometer()` | 加速度，m/s² |
| `Input.get_gyroscope()` | 角速度，rad/s |
| `Input.get_gravity()` | 重力向量，用于去重力 |

**Project Settings 必开：**

```
input_devices/sensors/enable_accelerometer = true
input_devices/sensors/enable_gyroscope = true
input_devices/sensors/enable_gravity = true
```

> Editor 桌面端返回 `Vector3.ZERO`，需真机或 Mock 调试。

#### 输出数据

| 参数 | 说明 |
|------|------|
| velocity | 速度 |
| angle | 出手角度 |
| spin | 旋转 |
| sideSpin | 横向旋转 |

---

### 3.2 动作识别模块（Throw Detection）

#### 核心难点

**如何判断「石头已扔出去」，但用户并没有真的把手机扔出去？**

真实打水漂时，石头离手后不再受手力；这里手机始终握在用户手中，不能依赖「设备离手」检测，必须从 **运动曲线** 推断虚拟出手时刻。

#### 设计原则

- 识别的是 **「模拟出手」**，不是检测手机脱手
- 用户全程握持手机，通过甩动轨迹的峰值与突变来锁定参数
- 误识别需有兜底：无效动作提示重试，避免误触发模拟

#### 功能

- 识别甩动开始（进入预备窗口）
- 识别 **虚拟出手瞬间**（锁定速度 / 角度 / 旋转）
- 锁定动作数据窗口，停止继续采集
- 触发实时物理模拟

#### 判定逻辑（V1 方案）

| 阶段 | 传感器特征 | 动作 |
|------|------------|------|
| 待机 | 加速度接近重力静止 | 等待甩动 |
| 甩动开始 | 加速度超过启动阈值 | 开始记录 1~2s 滑动窗口 |
| 虚拟出手 | 加速度达峰值后 **急剧回落**（甩动鞭打末端） | 锁定峰值时刻的参数作为「出手」 |
| 确认 | 角速度方向 + 水平分量符合投掷姿态 | 冻结数据，进入模拟 |
| 无效 | 未达阈值 / 窗口超时 / 姿态不符 | 提示「再试一次」，不进入模拟 |

#### 出手参数提取

- **速度**：出手前短窗口内加速度积分峰值
- **角度**：出手时刻设备 pitch（入水角）
- **旋转**：出手时刻 gyro 角速度（spin / sideSpin）

#### 校准与容错

- 阈值写入 `ThrowDetectionConfig.tres`（Resource），便于真机热更新
- 支持「练习模式」降低识别门槛，正式模式提高精度

---

### 3.3 简化物理模拟（Stone Physics Engine）

> 不使用真实流体模拟，不使用 `RigidBody3D` 流体；纯 GDScript 参数模型

#### 运行模式

| 模式 | 时机 | 输出 |
|------|------|------|
| **实时模拟** | 识别出手后立即运行 | 落点时间线 → 驱动震动；完成后输出 Skips / Score |
| **回放渲染** | 水漂结束后用户主动进入 | 读取已生成 Trajectory，播放 3D 动画 |

#### 核心逻辑

- 入水角 < 20° 且速度足够 → 触发水漂
- 每次水漂 → 速度衰减
- 超过阈值 → 下沉

#### 输出

- 水漂次数（Skips）
- 轨迹点（Trajectory）

---

### 3.4 3D 回放系统（Replay System）

> **定位**：水漂结束后的可选视觉重温，非主玩法必经步骤。

#### 触发条件

- 实时模拟完成、结果页展示后
- 用户点击「查看回放」进入
- 回放期间 **不触发震动**

#### 技术方案（Godot 4）

| 组件 | 用途 |
|------|------|
| `Node3D` + `MeshInstance3D` | 石头、水面 |
| `Camera3D` + 脚本 | 跟随 / 侧面镜头（无内置 Cinemachine） |
| `AnimationPlayer` / 协程 | 轨迹插值驱动 |
| `Engine.time_scale` | 慢动作回放 |
| **Mobile** 渲染器 | iOS / Android 3D 性能 |

#### 镜头类型

| 镜头 | 实现 |
|------|------|
| Follow | `Camera3D` 平滑跟随石头（`lerp` / `SpringArm3D`） |
| Side View | 固定侧面 `Camera3D`，切换 `current` |
| Slow Motion | `Engine.time_scale = 0.3` |

---

### 3.5 水面与特效系统（Water FX）

#### 特效

- 水花 Splash — `GPUParticles3D` 或 `CPUParticles3D`
- 波纹 Ripple — 圆环 Mesh + `Tween` 缩放淡出
- 泡沫 Foam — 可选粒子

#### 触发方式

- **回放模式**：每次石头接触水面时播放特效
- **实时模拟模式**：无画面（或极简 UI），仅按落点时间触发震动

#### 移动端注意

- 导出使用 **Mobile** 渲染器以支持 `GPUParticles3D`
- 老设备降级为 `CPUParticles3D`

---

### 3.6 震动反馈系统（Haptics）

> 震动是 **主玩法反馈**，发生在扔出之后的 **实时模拟阶段**，不在 3D 回放中重复。

#### 核心规则

**一个水漂落点 → 手机震动一次（实时，非回放）**

- 识别虚拟出手后，物理引擎按时间线计算各落点时刻
- 每个落点到达时触发 **1 次** 短震动
- 15 次水漂 = 15 次落点 = 15 次震动
- 3D 回放为纯视觉，**不再震动**

#### 触发点

| 时机 | 震动次数 | 说明 |
|------|----------|------|
| 虚拟出手确认 | 1 次 | 识别出手成功、开始实时模拟时 |
| 每个水漂落点 | 各 1 次 | 实时模拟中触水时刻 |
| 3D 回放 | 0 次 | 回放仅视觉 |

#### 反馈方式

- 单次短震（约 30–50ms），强度轻–中
- 不叠加、不连震
- 最后一次震动后短暂停顿，再展示结果页

#### Godot 实现

```gdscript
Input.vibrate_handheld(40, 0.5)  # duration_ms, amplitude (-1 = default)
```

| 平台 | 要求 |
|------|------|
| Android | 导出预设开启 **VIBRATE** 权限 |
| iOS | iOS 13+ 支持指定 `duration_ms` |

> V1.1 可选：GDExtension 接入 iOS Core Haptics / Android 精细波形。

---

### 3.7 评分系统（Scoring）

#### 输出

- 水漂次数
- 动作评分（0–100）
- 等级（S / A / B / C）

#### 示例

```
Skips: 15
Score: 88
Rating: A
```

## 4. 技术架构

```
┌─────────────────────────────────────────┐
│      Godot 4 客户端（iOS / Android）      │
├─────────────────────────────────────────┤
│  Input（传感器 + vibrate_handheld）      │
└─────────────────┬───────────────────────┘
                  ↓
      MotionCapture（_physics_process 采样）
                  ↓
    ThrowDetector（虚拟出手识别 ★难点）
                  ↓
      StonePhysicsEngine（纯脚本）
                  ↓
    HapticsScheduler（落点 → 震动，主反馈）
                  ↓
      TrajectoryStore（Resource / JSON）
                  ↓
           UI Layer（Control）
                  ↓
    ReplayScene（Camera3D + Particles，纯视觉）
```

## 5. 技术选型

### MVP 推荐

| 类别 | 选型 |
|------|------|
| 引擎 | **Godot 4.4+**（stable） |
| 语言 | **GDScript**（主） |
| 目标平台 | iOS + Android |

| 特效 | `GPUParticles3D` / `CPUParticles3D` |
| UI | Control + Theme |
| 配置 | `.tres` Resource |
| 构建 | 官方 Export Templates |

### 平台发布

| 平台 | 构建目标 | 分发渠道 |
|------|----------|----------|
| Android | Export → APK/AAB | Google Play / 侧载 |
| iOS | Export → Xcode 工程 | TestFlight / App Store |

### 平台权限与设置

| 平台 | 配置 | 用途 |
|------|------|------|
| Android | `VIBRATE` 权限 | `vibrate_handheld()` |
| Android | Sensors 开关（Project Settings） | 加速度计 / 陀螺仪 |
| iOS | macOS + Xcode 签名 | 导出与真机调试 |
| iOS | 传感器默认可用 | `get_accelerometer()` / `get_gyroscope()` |

### 与 Unity 版差异（选型参考）

| 项 | Godot 4 | Unity 6 |
|----|---------|---------|
| 传感器 | `Input` 轮询 | Input System Sensors |
| 镜头 | 自写 / 插件 | Cinemachine |
| 震动 | `vibrate_handheld()` | Handheld + 原生插件 |
| 包体 | 通常更小 | 通常较大 |
| 授权 | MIT 开源 | 商业许可 |

## 10. 风险

### 技术风险

| 风险 | 说明 |
|------|------|
| 虚拟出手识别 | **核心难点**：与引擎无关，需真机调参 |
| 传感器采样率 | Godot 无显式 100Hz 配置，依赖物理帧率轮询 |
| 跨平台一致性 | 双端投掷手感需统一校准 |
| Android 碎片化 | 低端机延迟高、震动弱；Godot 4.4+ 需开 VIBRATE 否则可能崩溃 |
| 移动端粒子 | Compatibility 渲染器不支持 GPU 粒子，需用 Mobile 或 CPUParticles |

### 产品风险

| 风险 | 说明 |
|------|------|
| 玩法疲劳 | 单一核心玩法可能导致用户留存下降 |
| 真实 vs 趣味 | 物理模拟精度与游戏趣味性之间的平衡 |

## 11. 一句话总结

用手机模拟投掷，Godot 识别虚拟出手后实时震出水漂节奏；水漂结束后，可选观看 3D 回放。
